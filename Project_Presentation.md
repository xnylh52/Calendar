## 第一部分：项目结构说明 (Project Structure)

本项目采用标准的 Android MVVM 架构，代码结构清晰，职责分明。

### 1. 核心代码目录 (`com.example.calendar2`)

*   **`MainActivity.kt`**
    *   **作用**：应用的主入口。
    *   **功能**：负责初始化日历视图（月视图/周视图）、展示日程列表、处理用户的点击事件（如选择日期、点击菜单），并与 ViewModel 进行数据交互。
*   **`AddEventActivity.kt`**
    *   **作用**：添加或编辑日程的页面。
    *   **功能**：提供表单让用户输入日程标题、地点、描述、选择时间、设置提醒，并将数据保存到数据库。
*   **`EventsAdapter.kt`**
    *   **作用**：日程列表适配器。
    *   **功能**：负责将日程数据渲染到主界面的列表（RecyclerView）中，处理“全天”事件的显示逻辑。
*   **`IcsUtils.kt`**
    *   **作用**：ICS 文件处理工具类。
    *   **功能**：实现了 iCalendar 标准的解析（导入）和生成（导出），支持跨平台日历数据互通。
*   **`LunarUtils.kt`**
    *   **作用**：农历算法工具类。
    *   **功能**：提供公历到农历的转换算法，用于在日历上显示农历日期和节气。

#### 子包说明

*   **`data/` (数据层)**
    *   **`Event.kt`**：实体类，定义了日程的数据结构（标题、时间、提醒设置等），对应数据库中的一张表。
    *   **`EventDao.kt`**：数据访问对象，定义了对日程表的增删改查（SQL）操作。
    *   **`AppDatabase.kt`**：Room 数据库的核心类，负责数据库的创建和版本管理。

*   **`viewmodel/` (业务逻辑层)**
    *   **`EventViewModel.kt`**：连接 UI 和数据的桥梁。它负责从数据库获取数据，并通过 Flow/LiveData 通知 UI 更新，同时处理后台任务（如保存数据），确保屏幕旋转时数据不丢失。

*   **`reminder/` (提醒功能)**
    *   **`ReminderManager.kt`**：提醒管理器，封装了 `AlarmManager` 的操作，用于设置和取消定时提醒。
    *   **`ReminderReceiver.kt`**：广播接收器，当设定的提醒时间到达时，它会接收系统广播并发送通知（Notification）给用户。

### 2. 资源文件目录 (`res/layout`)

*   **`activity_main.xml`**：主界面布局，包含日历控件（CalendarView）、切换按钮和日程列表。
*   **`activity_add_event.xml`**：添加日程界面的布局，包含输入框、日期选择器和保存按钮。
*   **`calendar_day_layout.xml`**：日历中“每一天”格子的布局，包含公历日期、农历日期和小圆点标记。
*   **`calendar_day_titles_container.xml`**：日历顶部的星期标题栏布局（如“一”、“二”...）。
*   **`item_event.xml`**：日程列表中单个日程项的布局，显示时间、标题和地点。
